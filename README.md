## Overview

| Property | Value |
|---|---|
| **Species** | *Saccharomyces cerevisiae* S288C |
| **Estimated Haploid Genome Size** | ~11.74 Mb |
| **Heterozygosity** | ~0.58% |
| **Sequencing Technology** | PacBio HiFi + BioNano Optical Maps + Hi-C |
| **Assembly Tool** | Hifiasm (HiC mode) |
| **BUSCO Lineage** | saccharomycetes_odb10 (n=2,137) |
| **Pipeline** | VGP on Galaxy |

The assembly pipeline follows four main stages: k-mer based genome profiling, phased contig assembly with Hifiasm, structural scaffolding with BioNano optical maps, and Hi-C contact-based scaffolding validation. The result is two fully phased haplotype assemblies (Hap1 and Hap2) with near chromosome-level contiguity, zero gaps, and >95% BUSCO gene completeness.

## Pipeline Stages

### Stage 1 — Genome Profiling

**Folder:** `01_genome_profiling/` | **Tools:** Meryl → GenomeScope 2.0

Before assembly, the genome was characterised by counting k-mers (k=31) from raw PacBio HiFi reads using **Meryl**, then fitting a statistical model to the histogram using **GenomeScope 2.0**.

#### What was done
Meryl processed raw HiFi reads to produce a k-mer frequency database and histogram. GenomeScope fitted a mixture of negative binomial distributions to estimate genome size, heterozygosity, repeat content, and sequencing coverage — entirely reference-free.

#### Results

| Parameter | Estimate |
|---|---|
| Estimated Haploid Length | 11,739,513 – 11,747,352 bp |
| Homozygosity (aa) | ~99.42% |
| Heterozygosity (ab) | ~0.58% |
| Repeat Content | ~723 kb (~6.2%) |
| Unique Sequence | ~11.02 Mb (~93.8%) |
| Model Fit | 92.5 – 96.5% |
| k-mer Coverage Peak | ~50× (heterozygous peak at ~25×) |
| Read Error Rate | 0.00094% |

#### Interpretation
The k-mer spectra shows a classic **bimodal diploid pattern**: a smaller heterozygous peak at ~25× (k-mers present in only one haplotype) and a dominant homozygous peak at ~50× (k-mers shared by both haplotypes). The very high homozygosity (~99.4%) is consistent with S288C being a heavily inbred laboratory strain. The excellent model fit (>92%) validates that the diploid model accurately describes this genome. The estimated size of ~11.74 Mb closely matches the known *S. cerevisiae* reference (~12 Mb including ribosomal regions).

> 📊 See `01_genome_profiling/genomescope/` for all four profile plot variants.

### Stage 2 — Hifiasm Phased Assembly

**Folder:** `02_hifiasm_phased_assembly/` | **Tools:** Hifiasm → gfastats → BUSCO → Merqury

#### What was done
HiFi reads were assembled into two fully phased haplotype sequences using **Hifiasm** (HiC mode), which leverages Hi-C chromatin contact information to assign contigs to their correct haplotype. Three QC tools were then applied: **gfastats** for structural statistics, **BUSCO** for gene-space completeness, and **Merqury** for reference-free accuracy assessment.

#### 2c — Merqury k-mer Quality Assessment

Merqury compares assembly k-mers back to the read k-mer database to measure completeness and base accuracy without any reference genome.

**Completeness**

| Assembly | k-mers Found | Total Read k-mers | Completeness |
|---|---|---|---|
| Hap1 | 11,611,483 | 13,010,260 | 89.25% |
| Hap2 | 10,792,811 | 13,010,260 | 82.96% |
| **Both combined** | **13,010,244** | **13,010,260** | **99.9999%** |

The combined completeness of **99.9999%** means both haplotypes together capture virtually every k-mer in the raw reads — the assembly is essentially complete.

**Error k-mers (base accuracy)**

| Assembly | Error k-mers | Total Assembly k-mers |
|---|---|---|
| Hap1 | 3 | 12,160,475 |
| Hap2 | **0** | 11,304,027 |
| Both | 3 | 23,464,502 |

Only 3 error k-mers exist across both assemblies. **Hap2 has zero QV errors**, confirming near-perfect base-level accuracy consistent with PacBio HiFi's very low error rate.

**Spectra Plots Explained**

The `merqury/plots/` directory contains two types of k-mer spectra:

- **`spectra-asm`** plots (`combined/`) show k-mers coloured by assembly origin: `read-only` (black — missing from assembly), `assembly_01-only` (red — Hap1-specific), `assembly_02-only` (blue — Hap2-specific), `shared` (green — in both). The dominant green peak at ~50× and minimal black signal confirm near-complete, accurate assembly with good haplotype phasing.

- **`spectra-cn`** plots show k-mer copy-number in each assembly: copy-number 1 (red) at ~50× represents correctly single-copy diploid k-mers; copy-number 1 at ~25× represents correctly assembled heterozygous k-mers. The near-complete absence of copy-numbers ≥3 confirms no spurious duplications.

> 📊 Three rendering styles per plot: `_fl` (filled), `_ln` (line), `_st` (stacked).

### Stage 3 — BioNano Hybrid Scaffolding

**Folder:** `03_bionano_scaffolding/` | **Tool:** BioNano Solve

#### What was done
BioNano optical genome mapping uses fluorescent labelling of enzymatic cut sites to generate ultra-long physical maps (up to several Mb). These maps were aligned to the Hap1 assembly to detect and resolve structural discordances and extend scaffold contiguity. The workflow aligns optical maps to contigs, identifies conflicts, makes targeted cuts where necessary, then merges the two data types into a hybrid scaffold.

#### Results

| Metric | BioNano Maps | NGS Input | Hybrid Scaffold |
|---|---|---|---|
| Count | 16 | 17 | 16 |
| Total Length | 12.030 Mbp | 12.161 Mbp | 12.075 Mbp |
| N50 | 0.922 Mbp | 0.923 Mbp | 0.923 Mbp |
| Largest | 1.530 Mbp | 1.532 Mbp | 1.532 Mbp |
| **Conflict cuts** | **0** | **0** | — |

#### Interpretation
**Zero conflict cuts** were required — the most important result of this stage. This means the Hifiasm assembly is in **perfect structural agreement** with the independently generated BioNano optical maps across all 16 scaffolds. No chimeric joins, no inversions, and no misassemblies were identified. The BioNano data served as independent orthogonal validation of the assembly's structural accuracy. The single unscaffolded NGS contig (85,850 bp) falls below the minimum optical map resolution threshold and is stored in the FASTA file of unscaffolded sequences.

> 📄 See `03_bionano_scaffolding/` for the scaffold report, AGP file (scaffold coordinates), and unscaffolded FASTA.

### Stage 4 — Hi-C Scaffolding & Contact Map Validation

**Folder:** `04_hic_scaffolding/pretext/` | **Tools:** PretextMap, PretextSnapshot

#### What was done
Hi-C chromatin conformation capture data records the 3D spatial proximity of genomic loci in the nucleus. When reads are mapped to the assembly and binned into a 2D contact frequency matrix, the result is a heatmap where contiguous sequence appears as a bright diagonal band. This provides chromosome-scale validation of scaffold order and orientation.

#### Results

Both haplotype contact maps display a **clean, uninterrupted diagonal** with strong signal concentrated along the main diagonal and no significant off-diagonal contacts.

#### Interpretation

| Observation | Meaning |
|---|---|
| Clean main diagonal | Correct scaffold ordering ✅ |
| No reversed segments | Correct scaffold orientation ✅ |
| No off-diagonal blobs | No chimeric/misassembled joins ✅ |
| Discrete diagonal blocks | Each scaffold = distinct chromosomal unit ✅ |
| Minimal background noise | High-quality Hi-C library ✅ |

This result is fully concordant with all previous QC metrics — zero BioNano conflicts, >95% BUSCO completeness, and near-perfect Merqury scores — providing multi-layered independent validation that the assembly is structurally correct at every scale from base pairs to chromosomes.

> 🖼️ See `04_hic_scaffolding/pretext/` for both haplotype full contact map images.
